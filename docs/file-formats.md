# File Formats

## Input

### Configuration file (`.txt`)

Plain-text file with space-separated key-value pairs. See [Configuration Reference](configuration.md) for all parameters.

```
outfile tutorial_single
n_bundles 1
lenght_side 30
icvf 0.3 0.75
```

### NFG strand files

External strand format (from the [NFG simulator](https://github.com/jonhrafe/MCDC_Simulator_public)). Convert to CACTUS format with:

```bash
cactus1-substrates convert -folder nfg_strands/ -outfile output.txt
```

## Pipeline outputs

### `.init` &mdash; Initial fibre placements (Stage 1)

Text file containing all fibre control points and radii before optimization.

**Format:** Each fibre is a sequence of lines with 4 values per point:

```
x y z radius
x y z radius
...
```

Fibres are separated by a header line containing the number of points.

### `optimized_final.txt` &mdash; Optimized fibres (Stage 2)

Same format as `.init`, but with fibre positions refined by the optimizer to minimize overlaps and match the target ICVF.

### `.partial` &mdash; Optimization checkpoints (Stage 2)

Intermediate fibre configurations saved every 50 iterations. Same format as `optimized_final.txt`. Useful for inspecting convergence or resuming from a known state.

### `.lconf` &mdash; Local configuration (Stage 1)

Auto-generated copy of the resolved configuration for each substrate. Records the exact parameter values used.

### Profiler logs (Stage 2)

| File | Content |
|------|---------|
| `profiler_ovlp.log` | Overlap metric per iteration |
| `profiler_non_ovlp.log` | Non-overlap metric per iteration |

Each line is one number: the metric value at that iteration. The optimizer stops when the overlap metric drops below `optimization_tolerance`.

### `.bz2` pickle files (Stage 3 &mdash; growth)

Compressed binary files containing the voxel-space representation of each fibre:

| File | Content |
|------|---------|
| `strand_XXXXX_dict.bz2` | Dictionary with metadata: bounding box, rotation matrix, box step, radius, color |
| `strand_XXXXX_mask.bz2` | Sparse 3D boolean mask of the fibre in voxel space |

### `.ply` &mdash; Triangle meshes (Stage 3 &mdash; meshing)

Standard [PLY format](https://en.wikipedia.org/wiki/PLY_(file_format)) triangle meshes generated by marching cubes. One file per fibre.

**Naming convention:**

```
strand_00042_outer_erode_0.ply
│       │      │      │
│       │      │      └── number of erosions applied
│       │      └── boundary type (inner/outer)
│       └── fibre index (zero-padded)
└── prefix
```

**Companion `.vol` files** contain the volume (in cubic micrometers) of each mesh.

### Combined mesh (paste-mesh)

The `paste-mesh` command merges individual fibre meshes into a single PLY file for simulation or visualization.

## Directory structure after a full run

```
tutorial_single_00000.init
tutorial_single_00000/
├── optimized_final.txt
├── profiler_ovlp.log
├── profiler_non_ovlp.log
├── 0_missing_axon_file.txt
├── *.partial
└── meshes/
    ├── pickles/
    │   ├── strand_00000_dict.bz2
    │   ├── strand_00000_mask.bz2
    │   └── ...
    ├── simulations/
    │   ├── strand_00000_outer_erode_0.ply
    │   ├── strand_00000_outer_erode_0.vol
    │   └── ...
    └── volume/
        └── ...
```
